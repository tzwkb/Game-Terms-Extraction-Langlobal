# 复盘：一次 Agent 误判与人工把关 —— "武茗" 案例

**日期**：2026-06-15  **项目**：古剑（古剑奇谭四）术语提取  **角色**：Agent 执行 / 人审

> 本文"Agent"特指**自主读写代码、跑流程、做删改决策并直接落地的编程智能体**，不是泛指的 AI/大模型。这次的危险恰恰来自它**自主执行**了删除——一个只给建议、由人动手的 AI 不会造成同样的破坏。

## 一句话

Agent 把一个**真实角色名**误判成"幻觉词"，还据此实现了一个**会静默删除真词**的过滤器并已应用；是人凭"三轮投票概率"的一句直觉把它拦了下来。

---

## 时间线

1. **v1 跑完**：Opus 4.8 抽出 248 个术语。
2. **Agent 质检**：发现 1 条 `武茗` 的 `Key值` 与 `来源原文` 两列空。
3. **Agent 初判（错）**：空源 = 原文里找不到这个词 = 模型幻觉 → 建议删除。
4. **Agent 危险修复（错）**：在 pipeline 加"空源即丢弃"过滤器，并已对 v1 应用，把 `武茗` 从 248 删成 247。
5. **人的关键一问**：
   > "这个词出现，是因为三轮投票都出现了幻觉吗？这概率太小了吧。"
6. **顺线查证 → 推翻**：去查源文件，真相大白（见下）。
7. **改对方向**：修匹配（而非删词）+ 过滤器改"标记不删"+ 恢复 `武茗`。

---

## 根因（查源文件后）

- 源文件 **第 560 行** `zh-Hans = "武  茗"`（"武"和"茗"之间有空格），`文本分类 = 角色名` → **它是真实角色名**。
- 模型把 `武  茗` 正确归一成 `武茗`，并在**三轮投票里稳定抽出**（识别完全正确）。
- Bug 在回溯匹配 `core/main.py::_batch_match_context`：用**精确子串** `"武茗" in "武  茗"` → 因空格判 `False` → 源挂不上 → 两列空。

**结论：所谓"空源"不是幻觉，是匹配对空格不容忍造成的假阴性。**

---

## Agent 错在哪（两处，都写明）

1. **误判**：把"空源"直接等同"幻觉"，**没去查源文件**就下结论——一段听起来合理的叙述（"模型在一串名字里脑补了个武侠名"），整个是错的。
2. **危险修复**：基于错误前提做了"空源即删"的过滤器。它会**静默删除像 `武茗` 这种带格式空格的真词**——是数据破坏，且不可逆。Agent 提出的"修复"比原问题更危险。

## 人对在哪（一处，决定性）

- 抓住**三轮投票的统计含义**：系统 `cnt≥3` 才保留。若 `武茗` 是幻觉，等于三轮独立生成了**同一个生造名**，概率极低。
- 所以"它能留下来"本身就是"它是真的"的证据。这一步直接推翻了 Agent 的幻觉假设，把方向从"删词"扳回"查源 + 修匹配"。

---

## 正确修复（本次同批提交）

| 项 | 改法 |
|---|---|
| `_batch_match_context` | 子串匹配前对**两边归一化去空白**（`\s+`→空），`武茗` 正确挂回 560 行 |
| 未命中源的词 | 从"**删除**"改为"**标记**"：`备注` 打 `⚠原文未命中,待核`，**绝不删**（因为可能是真词） |
| `PipelineOpts` / CLI | `drop_unsourced` → `flag_unsourced`（默认开）；`--no-flag-unsourced` 可关 |
| v1 产物 | 恢复 `武茗`（248 条），Key=`Table,FNpc_52900000_Name`，来源=`武  茗` |

**修订后真实幻觉率：0 / 248**（Opus 一个都没编）。

---

## 教训

1. **三轮投票的真正价值**：不仅过滤高频噪音，更给出"**存活 = 可信**"的信号——一个词能扛过 3 轮，通常是真的，不是假的。读结果要会用这个信号。
2. **"空源 ≠ 幻觉"**：表象有多个可能成因（这里是源文本的内部空格）。下结论前先**查原始数据**。
3. **Agent 的合理叙述可能整体是错的**：越像模像样越要警惕。涉及**删除/不可逆**的动作尤其要先验证、再由人确认。
4. **Agent 的"修复"可能比问题更危险**：一个自动删数据的过滤器，错误前提下就是定时炸弹。
5. **人在回路不是形式**：领域直觉（"概率太小"）正是 Agent 容易跳过、而人一眼能察觉的那层校验。**Agent 编程提速，但拍板和兜底仍离不开人。**
